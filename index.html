인생 로드맵: 사람을 위한 기술로

신학을 깊이 공부하며 깨우친 인문학적 가치관이 데이터를 만난 세 번의 마일스톤입니다.

인문학적 고민
Turning Point 01

길 잃은 신학도, 봉사활동에서 길을 찾다

원래 전공은 신학이었습니다. 군 복무와 휴학 중 참여했던 사회공헌 봉사활동에서 어르신들과 취약계층 분들이 교묘한 금융 사기 피해로 눈물 흘리는 모습을 마주했습니다. 기술이 사람에게 비수가 아니라 실질적인 울타리가 될 수 없을까? 이 뼈저린 고민이 제 가치관의 뿌리가 되었습니다.

데이터와의 조우
Turning Point 02

"데이터는 사람의 행동을 설명하는 언어다"

진로를 전환하며 관심 가졌던 마케팅 대외활동(청년이룸 등)에서 구글 애널리틱스(GA)를 접했습니다. 화면 안 무수히 많은 지표들이 단순히 숫자가 아니라, 웹을 이용하는 진짜 사용자의 '한숨', '머무름', '망설임'이라는 것을 깨닫고 본격적인 데이터 공부의 노를 저어 가기 시작했습니다.

원리의 정면 돌파
Turning Point 03

한계를 딛고, 원리를 파고드는 엔지니어로

신용카드 이탈 예측 프로젝트 당시, 정밀하게 짠 머신러닝의 내부 수학 원리를 제대로 설명하지 못해 침묵해야 했던 뼈아픈 한계를 마주했습니다. 도구만 휘두르는 것이 아니라 원리를 100% 장악하고 설명할 수 있어야 한다는 집념으로 저는 단순 분석가에서 한 단계 더 나아가 데이터의 처음과 끝을 설계하는 엔지니어 영역에 도전하게 되었습니다.

나의 벽을 마주하며 쌓은 벽돌들

"저는 결코 처음부터 완성된 인재가 아닙니다. 다만 막히는 모든 한계를 정면 돌파하며 외연을 견고히 확장해왔습니다."

마케팅 대외활동

마케팅 데이터(GA)로 사용자의 갈증에 눈뜨다

처음 접했던 데이터의 역할은 '사용자 심리의 대변자'였습니다. 이탈 고객들이 어떤 버튼에서 멈칫했는지 가설을 도출하며 데이터 분석에 깊은 동기부여를 받았습니다.

Google Analytics 검색광고마케터 1급 사용자 행동 파악

마주한 벽과 해결법

"분석 도구가 던져주는 한정된 리포트의 아쉬움"

GA 화면 상의 단순 UI만으론 세밀한 행동 패턴을 밝힐 수 없었습니다. 이를 계기로 원시 로그 데이터를 직접 만질 수 있는 파이썬과 SQL 학습에 올인하였습니다.

데이터 마케팅 & 기본기

SQL과 EDA로 '비즈니스의 흐름'을 설계하다

단순 쿼리 작성을 넘어, 가설을 수립하고 비즈니스 지표(AARRR 등)를 명확하게 세워나가는 법을 익혔습니다. ADsP, SQLD를 공부하며 데이터 마트의 중요성을 직접 깨달았습니다.

SQLD & ADsP AARRR 지표 설정 Tableau 시각화

마주한 벽과 해결법

"지표 너머, 실제 원인은 어떻게 예측할 것인가?"

흘러간 과거 지표 정리만으로는 다가올 이탈 고객의 행동을 원천 방어할 수 없었습니다. 더 깊은 예측을 위해 머신러닝 이론으로 깊숙이 파고들었습니다.

머신러닝 예측 프로젝트

예측을 하며 '이유를 설명하는 능력'의 중요성을 배우다

신용카드 이탈 예측 프로젝트를 진행하며 모델의 정확도 뿐 아니라, 도메인에 대한 직관을 보태기 위해 CatBoost와 Kmeans Clustering을 유기적으로 엮어 실제 타겟 마케팅 방안(LTV 기반 약 4.4억원 절감 효과)을 도출했습니다.

CatBoost & Optuna K-Means & RobustScaler SHAP Feature Importance

마주한 벽과 해결법

"모델링의 수치 조정보다 중요한 것은 탄탄한 파이프라인"

모델 파라미터 튜닝에 밤새워 매달리던 중, 로컬 주피터 환경의 모델은 실제 현업의 지속적인 비즈니스 가공 과정(Raw ➔ Staging ➔ Mart)을 보조해주지 못한다는 전반적 한계를 느끼고 아키텍처에 관심을 키웠습니다.

데이터 엔지니어링 확장

데이터 파이프라인과 배포로 '처음부터 끝'을 뚫다

분석가에만 가둬져 머무르고 싶지 않아, 원본 데이터를 Raw ➔ Staging ➔ Data Mart로 나누어 설계하는 파이프라인을 구축했습니다. 파이썬 기반 데이터 적재를 자동화하고 사용자들이 업로드하면 Streamlit을 통해 누구나 가치를 볼 수 있게 실배포까지 마무리 지어 보았습니다.

PostgreSQL & PySpark E2E Pipeline Streamlit & Cloud 배포

마주한 벽과 해결법

"견고하지 못한 코드 구조와 정밀도 아쉬움"

파이프라인 구조를 짜면서 기초적인 컴퓨터 시스템 지식, 그리고 대용량 처리를 위한 단단한 자료구조와 알고리즘의 실전 부재를 뼈저리게 느꼈습니다. 이것이 취업보다 1년의 훈련(SSAFY)에 도전하기로 결심한 결정적 한 발자국입니다.

SSAFY 1년의 다짐
